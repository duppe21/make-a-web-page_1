Lesson 1: The Basics Web and HTML

How does the Web work?

Developed in the early 1990's the web is made up of many different components.

Major pieces of the web are;
YOU: We know who this is
A Computer & Web Browser: You computer running Internet Explorer, FireFox, Google Crome or any other web browser
The Internet: www or World Wide Web
Servers: Servers are machines similar to your computer but their main functions are to host files
HTTP: This protocol is used by your web browser to communicate requests to the servers over the internet
HTML: Hypertext Markup Language is a set of standards used to tag the elements of a hypertext document. It is the standard protocol for formatting and displaying documents on the web.

You type in a URL to your computers web browser requesting webpage. Your computer sends your request over the internet using HTTP "Hypertext Transfer Protocol" to the server where you webpage is stored. The server responds by sending files that your browser displays.

HTML

HTML "HyperText Markup Language" is the basis for almost every webpage. HTML documents are the heart of the web. The links between these pages are called hyper-links.

HTML is Made Up of;

  1. Text Content - This is what you see
  2. Markup - This is what it looks like or how its arranged
  3. Reference to other documents- images, video etc.
  4. Links to other pages
HTML Markup

HTML is made up of things called TAGS. A tag consists of an opening tag, content and a closing tag. This tag structure is called an Element. The contents of the elements can be more tags, attributes, text, links or images.

Some TAG samples can be found HERE

b tag = BOLD Everything within the opening and closing b tag will be BOLD
em tag = Emphasis Everything within the tag will be in italics

Important to keep in mind. If you don't close your tag around your desired content your entire document will be as your defining tag.
Make sure to close your tag!

Whitespace

All whitespace in HTML regardless of how it looks to you in the editor turns into a single space. You'll need to define new lines or blocks.

br tag = stands for BREAK and is also a void tag. This simply creates a break in a line and creates a new one.

p tag = Paragraph This is not a void tag because it has content associated with it. It defines a block on information.

Inline vs. Block

HTML elements are either inline or block.
Inline: The br tag is an example of an inline tag. It simply just ends a line. Inline tags just deal with line content.
Block: The p tag is an example of a block tag. When used it defines a block of content. It creates an invisible block around your content.

Container Elements
The "span" and "div" tags do nothing but contain their content. You can add style to them to adjust the way the content looks.
span = (Inine) tags contain content within a line or a single element.
div = (Block) tags contain a block of content that can be any number of elements. The div tag plays a huge roll in helping set the structure of you page.

Lesson 2 Creating a Structured Document with HTML

HTML Tree-Like Structure

"DOM" When programmers talk about (Document Object Model) they are talking about the tree-like structure of a page that can be found in the develpers tools.The browser builds a tree like structure of the page of HTML. You can see this by going to your browser tools and selecting the developer tools.

A good place to learn more about the Document Object Model is this wikipedia artical on Tree Structure.

In the devloper tools you'll see a window with lines of HTML and sidways triangles. You can scoll over the lines and it will highlight that area on your webpage.
Take a look at the sideways triangles. These identify different elements. These elements can contain other elements. You can click on the triangle to open the element to see a cascade of other elements and tags. This gives you the "TREE" also called the DOM tree.

A couple lessons learned so for. Giving your class a meaningful name and having good structure will make thinks easier on you. Being able to read through and interpret your code easily will help you locate and correct errors that much faster.

Lesson 3 Adding CSS Style and HTML Structure

What is CSS?

Cascading Style Sheet or CSS is code written to control the "style" of HTML elements.

HTML and CSS are both languages
HTML is the language of the web. It contains syntax and rules for controling the structure of a web page
CSS is the style. It controls how the webpage looks.

Repetition
Avoiding repetition can be key to successful programming. By continually repeating or rewriting the same code or style attributes over and over throughout your document you can cause errors if you need to make changes. By using CSS you can assign style to your div class once. This way each section with the same div will be the same and when you want to make changes you can be sure that the change will be made in each desired section.
It didn't take me long to figure this one out and realize the benefit of CSS with being able to assign style throughout the document by making changes in one location. Each div class can be changed from the CSS keeping you from having to skim through the document hoping you've made all the changes the same.

There are three ways you can apply CSS styles;

  1. Apply Directy to the HTML element
  2. Embed within the HTML document head
  3. Create and use an external style sheet

Using options 2 and 3 are useful in helping you avoid repetitive entries.
